BaseServerConfig

Interface of the base server config object which defines the following properties:

  • port
  • timezone
  • predictive404
  • predictive404Priority
  • logRequestHeaders
  • excludeHeadersInLogs
  • excludeQueryParamsInLogs
  • colorfulLogs
  • consoleLogLevels
  • writeLogsToFile
  • logFileLevels
  • logFileMaxAge
  • archiveLogs
  • fileUploadLimit
  • sessionManagement
  • cookieSecret

port

Type: number
Required: No
Default value: 5000
Description: The port number to run the server at.

// src/config.ts
export const config = {
  port: 5000
};

timezone

Type: string
Required: No
Default value: Local system timezone
Description: The timezone of the server which can be any IANA zone supported by the host environment, or a fixed-offset name of the form 'UTC+3'.

// src/config.ts
export const config = {
  timezone: 'America/Los_Angeles'
};

predictive404

Type: boolean
Required: No
Default value: false
Description: If true, installs a 404 handler at the top of the stack instead (this can be configured through predictive404Priority), which predicts if path will match with any middleware in the stack and rejects the request with a 404 error if not. This is useful as requests that will eventually result in a 404 error won't go through the stack in the first place. Please note that the prediction ignores all * and / routes.

// src/config.ts
export const config = {
  predictive404: true
};

predictive404Priority

Type: number
Required: No
Default value: Infinity
Description: The priority of the predictive 404 middleware.

// src/config.ts
export const config = {
  predictive404Priority: Infinity
};

logRequestHeaders

Type: boolean
Required: No
Default value: false
Description: Indicates whether request headers should be logged for each request.

// src/config.ts
export const config = {
  logRequestHeaders: true
};

excludeHeadersInLogs

Type: Array<string>
Required: No
Default value: ['authorization']
Description: A list of header names to hide in logs when logging request headers (controlled by logRequestHeaders). Useful for hiding sensitive information from logs.

// src/config.ts
export const config = {
  excludeHeadersInLogs: ['user-key']
};

The authorization header is the default value and cannot be overridden. It will be pushed to any array passed as the config value.

excludeQueryParamsInLogs

Type: Array<string>
Required: No
Default value: []
Description: A list of query parameter names to hide in logs. Useful for hiding sensitive information (tokens, etc.).

// src/config.ts
export const config = {
  excludeQueryParamsInLogs: ['token']
};

colorfulLogs

Type: boolean
Required: No
Default value: true
Description: Indicates whether console logs should have color.

// src/config.ts
export const config = {
  colorfulLogs: false
};

consoleLogLevels

Type: 'all' | Array<'debug'|'info'|'notice'|'warn'|'error'>
Required: No
Default value: ['info', 'notice', 'warn', 'error']
Description: An array of log levels to show in the console or all (no array) to show all levels.

// src/config.ts
export const config = {
  consoleLogLevels: 'all'
};

writeLogsToFile

Type: boolean
Required: No
Default value: true
Description: Indicates whether logs should be written to file.

// src/config.ts
export const config = {
  writeLogsToFile: true
};

logFileLevels

Type: 'all' | Array<'debug'|'info'|'notice'|'warn'|'error'>
Required: No
Default value: all
Description: An array of log levels to write on disk or all (no array) to write all levels when file logging is enabled.

// src/config.ts
export const config = {
  logFileLevels: ['debug', 'notice', 'error']
};

logFileMaxAge

Type: number
Required: No
Default value: 7
Description: The maximum age of a logs file in days. When passed, the file will either get archived or deleted depending on archiveLogs.

// src/config.ts
export const config = {
  logFileMaxAge: 1
};

archiveLogs

Type: boolean
Required: No
Default value: true
Description: Will archive logs written on disk that are older than their max age instead of deleting them.

// src/config.ts
export const config = {
  archiveLogs: false
};

fileUploadLimit

Type: string
Required: No
Default value: 10mb
Description: The file upload limit with suffix accepted by the server (e.g. kb, mb, gb).

// src/config.ts
export const config = {
  fileUploadLimit: '2mb'
};

sessionManagement

Type: boolean
Required: No
Default value: false
Description: Enables session management through cookies and ServerSessionManager.

// src/config.ts
export const config = {
  sessionManagement: true
};

cookieSecret

Type: string
Required: No
Default value: undefined
Description: A secret to sign cookies with. Cookies will be unsigned if no secret is provided. If session management is enabled (controlled by sessionManagement), all session cookies will be signed with this secret (if provided).

// src/config.ts
export const config = {
  cookieSecret: process.env.COOKIE_SECRET
};
